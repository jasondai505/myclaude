## 0. 统一调度器（中枢）

**文件**: `orchestrator.py`  
**触发**: Windows 计划任务 `myclaude_evening` (17:50) + `myclaude_morning` (6:30)

**五段流水线时序**：

| 流水线 | 触发时间 | 跳过条件 | 内容 |
|--------|:------:|------|------|
| **close** | 17:50 | 休市日自动跳过 | 复盘 + 催化走势确认 + 日报简报 |
| **night** | 17:50（close 之后） | — | `daily_collect --tier daily`（全源采集+所有deep_read）+ 公众号分析 + 星球分析 + 四源交叉验证 + 催化筛查 + 产业链更新 |
| **pre_dawn** | 6:30 | — | 美股收盘数据 + FEV/G-Factor/Δ 刷新 + 边际变化检测 |
| **pre** | 6:30（pre_dawn 之后） | — | 盘前解读 + 盘前建议生成 + 启动 Advice HTTP 服务 |
| **pre_game** | 9:00 | — | 星球/微博增量刷新 → 守门员第二份建议 |

**close 内部步骤**：健康检查 → 收盘复盘(`run.py`) → 催化跟踪(`daily_collect --tier post_market`) → 日报简报 → 输出自检

**night 内部步骤**（12步）：看门狗 → 健康检查 → 日更采集(全源+所有deep_read) → 周更(仅周五) → 公众号AI分析 → 星球AI分析 → 复盘摘要 → 推荐追踪 → 四源交叉验证 → 催化筛查 → 产业链更新 → 输出自检

> **注意**：`daily_collect --tier daily` 默认回补 7 天。所有 deep_read collector 已修复为遍历完整日期范围 + 幂等跳过已处理日期。交互易采集超时 1200s（88只逐只调 akshare）。

## 1. 公告 LLM 五维评分

**文件**: `daily_review/deep_read/llm_deep_read.py`  
**成本**: ~$9/天 (Haiku 提取 + Sonnet 评分)  
**流程**: Stage1 硬筛选（猎场漏斗）→ Stage2 Haiku 结构化提取 → Sonnet 五维评分

| 维度 | 满分 | 判断内容 |
|------|:---:|------|
| 核心矛盾 | 40 | 是否触及企业长期自由现金流创造能力？第二增长曲线/护城河质变/致命风险解除 |
| 信息增量 | 20 | 公告带来的新信息量？已知信息的重复 vs 全新的战略动作 |
| 利益一致性 | 15 | 内部人行为与公告方向是否一致？增持/认购定增 = +，减持 = - |
| 治理信号 | 10 | 监管问询/处罚/诉讼 vs 股权激励/回购 |
| 场景校准 | 15 | 当前市场环境下该公告的敏感性？牛市中被忽略的利好 vs 熊市中被放大的利空 |

**总分 0-100**，≥60 高价值。阈值定义在 `config.DEEP_READ_RULES.min_deep_read_score`。

## 2. 催化四维评分

**文件**: `daily_review/catalyst_screen.py`  
**公式**: `行动性 = M×3 + S×2 + N×2 + U×1`（8-80 分）

| 维度 | 权重 | 定义 |
|------|:---:|------|
| Magnitude（幅度） | ×3 | 对基本面/供需/价格的影响有多大？ |
| Specificity（具体性） | ×2 | 是可验证的具体事件还是模糊传闻？ |
| Novelty（新奇度） | ×2 | 第一次出现还是老调重弹？ |
| Urgency（紧迫性） | ×1 | 需要立刻行动还是可以慢慢观察？ |

**机械加分项**（独立于 LLM）：
- 多源确认 +5
- 供给冲击/涨价 +3
- 减持需带比例判断：<0.1%=噪声, 0.1-1%=-1~-2, 1-3%=-3~-4, >3%或大股东=-5~-7

## 3. 研报档案星级评分

**文件**: `daily_review/_dashboard.py → _score_dossier()`  
**用途**: 对 `research_dossiers/*.md` 中的个股档案计算 1-5 星级

| 维度 | 满分 | 计算方式 |
|------|:---:|------|
| 信号强度 | 30 | 绝对值加总 ×2（调研密集+3, 互动关键词+3, 业绩暴增+5 等） |
| LLM 评分 | 30 | Sonnet 研报分析得分 ×0.3 |
| 机构覆盖 | 15 | 评级表中机构行数 ×3 |
| 信号频率 | 15 | 不同信号日数 ×5 |
| 新鲜度 | 10 | 当天+10, 3天内+6, 7天内+3, 14天内+1 |

| 星级 | 分数 |
|:----:|:---:|
| ★★★★★ | ≥70 |
| ★★★★ | ≥50 |
| ★★★ | ≥30 |
| ★★ | ≥15 |
| ★ | <15 |

## 4. Dashboard 多层架构

**文件**: `daily_review/_dashboard.py`  
**更新频率**: 每日（`daily_collect --tier daily` 末尾）  
**7 层结构**：

| 层 | 段落 | 函数 | 说明 |
|:--:|------|------|------|
| 1 | 🎯 值得关注的标的 | `_aggregate_signals()` | 6 源信号复合评分，含链信息（板块/层1/层2） |
| 2 | 🔥 信号热度 TOP 主题 | `_hot_themes()` | 四源交叉主题热度 |
| 3 | 🔗 逻辑×走势双轨验证 | `_chain_heat()` 增强 | 链段级 共振/预期差/走势先行 三分类 |
| 4 | 📅 主题生命周期 | `engine_theme_lifecycle.py` | 主题时间轴+状态机，emerging→active→confirmed→cooling |
| 5 | 🧠 主题建议 | `_render_theme_advisor()` | 规则引擎交叉生命周期×预期差，零 LLM |
| 6 | 🏆 双轨选股 | `dual_track_screener.py` | FEV轨 + G-Factor轨分席输出 |
| 7 | 📈 盘面概念信号 | `_render_price_signals()` | 概念热度+持续性板块+相似日预测 |

### 4a. 6 源信号聚合（第 1 层）

**文件**: `_aggregate_signals()`  
**方法论**: 见 [CLAUDE.md](../CLAUDE.md#dashboard-6-源信号聚合-critical--任何信息源权重调整前必读)

**6 源及权重**：

| # | 源 | 单位 | 权重 | 分值范围 | 数据来源 |
|---|----|------|:---:|:------:|------|
| ① | 公告 deep_read | score（10~85） | ×1.0 | 10~85 | `deep_read_results` 表 |
| ② | 研报 | 篇数 | ×20 | 0~N×20 | `research_reports` 表 |
| ③ | 新闻边际信号 | 条数 | ×8 | 0~N×8 | `feeds/news_signals/*.md` |
| ④ | 行业深研提及 | 次数 | ×10 | 0~N×10 | `reports/industry/*.md` |
| ⑤ | 社交源提及 | 次数 | ×10 | 0~N×10 | 公众号分析 + 微博 feed |
| ⑥ | 催化信号 | actionability | ×0.8 | 0~64 | `catalyst_signals` 表 |

**交叉源加分**：

| 覆盖源数 | 加分 |
|:-------:|:---:|
| 2 | +10 |
| 3 | +20 |
| 4 | +35 |
| 5 | +50 |
| 6 | +65 |

**分级阈值**：

| 级别 | 阈值 | 含义 |
|------|:---:|------|
| 🔬 深度分析 | ≥80 | 多源强信号或单一源极高强度 |
| 👀 初步关注 | 20~79 | 1 篇研报或 2 次社交提及即达标 |

**设计原则**：研报和社交源是重点资源（权重 10-20/次），公告已通过 Stage1 硬筛选（至少 10 分），新闻经 Haiku 扫描已过滤无关内容。所有源不做二次过滤，靠复合分排序。

## 5. 新闻边际信号检测

**文件**: `daily_review/collectors/news_signals.py`  
**成本**: ~$0.01/天 (Haiku 批量扫描)  
**流程**: 读取 `news_{date}.md` → 分批 Haiku 扫描 → 提取边际变化

**边际变化定义**（满足任一）：
- 订单/合同/中标、产品涨价/降价/产能变化
- 政策利好/利空、业绩预告/修正
- 技术突破/新产品/新产线、大股东增减持/回购/定增/重组
- 行业供需格局变化

**排除**：股价涨跌、例行公告、重复报道、机构调研、纯概念炒作

每条信号带 `direction`（利好/利空/中性）和 `confidence`（高/中/低）。

## 6. 研报跟踪

**文件**: `daily_review/collectors/research_deep_read.py`  
**数据源**: 东方财富全市场 API → `research_reports` 表  
**成本**: ~$1/天 (Sonnet 分析)

**流程**:
1. 全市场研报采集（~75 篇/天）→ `research_reports` 表
2. 信号检测：`detect_signals()` 机械规则识别 → 首次覆盖/评级变化/EPS 修正/密集覆盖
3. LLM 分析：有信号的个股 → Sonnet 生成投资逻辑（200-300 字）+ 评分 + 时间跨度
4. 存档：写入 `research_dossiers/{code}.md` 累积档案（调研+互动易+业绩信号也追加到同一档案）

**与 Dashboard 的关系**：研报原始采集量（75 篇）直接纳入 6 源复合评分。LLM 分析后的档案星级（1-5★）通过 `_score_dossier()` 计算，但目前 Dashboard 未直接引用星级（仅通过 `_aggregate_signals` 中的 `rpt_count` 体现）。

## 7. 社交源

**数据源**: 微信公众号（WeWe-RSS）+ 唐史主任微博 + 知识星球 + 韭研脱水研报  
**成本**: ~$1-2/天 (Haiku 分析)

| 源 | 采集器 | 产出 | LLM | 特点 |
|----|------|------|:--:|------|
| 微信公众号 | `wechat.py` | `feeds/wechat/*.md` + `wechat_analysis/*.md` | ✅ Haiku+Sonnet 两阶段 | 200篇/天拉取，精选行业大V |
| 唐史主任微博 | `weibo.py` | `feeds/weibo/*.md` | ✅ Haiku 方向+标的提取 | 顶级宏观+产业审美，~6帖/天 |
| 知识星球 | `zsxq.py` | `feeds/zsxq/*.md` + `zsxq_analysis/*.md` | ✅ Haiku+Sonnet 两阶段 | ~60帖/天，催化信号主来源 |
| 韭研脱水研报 | `jiuyang.py` | `feeds/jiuyang/*.md` | ❌ 仅 PDF 文本提取 | 用户生成内容，质量不稳定 |

**与 Dashboard 的关系**：社交源提及的股票代码直接纳入 6 源复合评分（10 分/次）。公众号和星球的两阶段 AI 分析产出主题摘要，供 `_hot_themes()` 做跨源主题热度排名。

> **微博 Cookie 维护**：微博 API (`weibo.com/ajax/statuses/mymblog`) 需有效 Cookie，过期后 collector 返回 "no posts" error。更新方法：浏览器登录 → F12 Network → 复制 Cookie → 更新 `morning_intel/settings.py` 的 `WEIBO_COOKIE`。

## 8. 主题热度排名

**文件**: `daily_review/_dashboard.py → _hot_themes()`  
**四源交叉**: 催化信号 + 行业深研 + 公众号分析 + 星球分析

**关键词映射** (KW_MAP): 45 个核心产业关键词（PCB/MLCC/AI芯片/GPU/HBM/算力/存储/液冷/光模块/CPO/先进封装/半导体设备/碳化硅/覆铜板/断供/缺货/涨价/产能扩张/新能源/锂电/光伏/储能/机器人/军工/航天/邮轮/VLCC/航运/稀土/收购/重组/跨界/技术突破/业绩暴/股权激励/减持/华为/特斯拉/比亚迪/宁德）

**热度计算**: `信号数 // 5 + 1` (上限 3 星)  
**来源标签**: 催化 / 行业深研 / 公众号 / 星球
