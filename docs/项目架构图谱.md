## 二、每日流水线时间线

```
06:30  pre_dawn   ─┬─ FEV/G-Factor 评分刷新
                   ├─ Δ 边际变化评分
                   └─ 星球增量分析
07:30  pre        ─┬─ 盘前建议生成 (第一份)
                   └─ advice_server 启动
09:00  pre_game   ─┬─ 星球 + 微博增量同步
守门员              ├─ 公众号增量分析
                   ├─ 美股数据补全 (如 07:30 未拉到)
                   └─ 盘前建议刷新 (第二份)
10:30  intraday   ─┬─ 快扫描 (2min) + 深扫描 (15min)
~15:00              └─ 双频自循环, 休市自动退出
17:00  close      ─┬─ 复盘报告 (12 阶段引擎)
                   └─ 催化跟踪 + 晨间简报
22:00  night      ─┬─ 18 源全量采集
最重                ├─ 两阶段 LLM: 公众号 + 星球
                    ├─ 四源交叉验证 (primary_synthesis)
                    ├─ 催化筛查 + 标的映射
                    ├─ 行业深研 + 产业链知识库
                    ├─ 仪表盘 + 索引页 生成
                    └─ 🔒 执行锁 | ⛓️ 失败阻断 | 🗑️ 日志轮转
```

### night 流水线 depends_on 依赖链

```
health → watchdog → collect ─┬─→ wechat (公众号分析)
                             ├─→ synthesis (四源交叉)
                             └─→ catalyst_screen (催化筛查)
       collect 失败 → 后续 3 步自动 SKIP，不浪费 LLM 费用
```

## 十、核心方法论索引 (详见 CLAUDE.md)

| 铁律 | 要点 |
|------|------|
| **防幻觉** | 注入覆盖 + 代码全量提取 + 输出逐条校验 |
| **名称→代码双通道** | 正则 6 位代码 + 全名反向匹配，缺一不可 |
| **输入审计** | 新管线上线必打印完整 prompt 人肉确认 |
| **降级链完整性** | 每个调用点传齐所有降级参数 |
| **LLM 输出校验** | 代码白名单 + 名称匹配 + 行业一致性 |
| **防混锅** | 富集比检验 ER < 0.8 = 噪音；单主因归因 |
| **信息→行动断点** | 采集→筛选→深度→映射→跟踪→建议 |
| **催化四维** | M×3+S×2+N×2+U×1 机械合成 |
| **双轨排名** | 不可公度维度不强行加权，分席位 |
| **盘中阈值降级** | timeliness > certainty |
| **时间窗口合并** | 昂贵做增量 (LLM 只跑一次)，便宜做存量 (DB+Redis) |
