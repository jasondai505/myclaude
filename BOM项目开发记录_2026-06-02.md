# BOM 产业链分析工具 — 开发全记录

**日期**：2026-06-02  
**触发**：用户提供博主「廖小爱」的 BOM 拆解方法论，要求工具化

## 一、项目概述

将博主「用 BOM 拆解法选股」的方法论，从「手动问 LLM」升级为自动化工具。核心链路：

```
输入行业 → BOM产业链拆解 → 筛三高赛道 → 锁A股龙头 → 验护城河 → 生成报告
```

### 最终交付

```
bom_analyzer/
├── run.py              # CLI 入口（argparse 两阶段管线）
├── config.py           # 阈值/路径/LLM 参数
├── models.py           # BomSegment / HighValueSegment / LeaderStock / MoatScores
├── chain_db.py         # SQLite 知识库（3表）+ 行业快照
├── report.py           # Markdown 5节报告渲染
├── prompts/
│   ├── stage1_chain.txt    # 拆产业链 + 筛三高
│   └── stage2_leaders.txt  # 锁龙头 + 验护城河
├── reports/            # 7份报告已生成
└── data/
    └── bom.db          # 产业链知识库
```

## 二、沟通过程与关键决策

### 决策 1：两阶段 vs 四阶段

博主方法论是 4 步（拆产业链 → 筛三高 → 锁龙头 → 验护城河）。

**决策**：合并为两阶段。
- Stage 1（拆产业链 + 筛三高）— 天然连贯，无需中间等待
- Stage 2（锁龙头 + 验护城河）— 需要真实财务数据支撑

理由：4 次 API 调用变 2 次，成本减半，Step 1+2 和 3+4 紧密耦合。

### 决策 2：数据注入模式

延续 `_run_advice.py` 的 `%%PLACEHOLDER%%` 替换模式。Prompt 模板只放指令框架，数据由 Python 预先拉取注入。**严禁在 prompt 中写「读取文件」类指令**（参考 2026-05-29 幻觉事故）。

### 决策 3：防幻觉机制

Stage 2 输出的股票代码，用腾讯行情 API 逐条校验代码-名称是否匹配。不匹配的自动修正并打 ⚠️ 标记。

### 决策 4：财务数据补全

初版 PE/ROE/CAGR 全为 0（Stage 1 代表公司多为海外企业，A 股匹配失败）。

**修复**：Stage 2 拿到 A 股代码后，加一步 `_enrich_financials()`：
- 腾讯行情 API → PE/PB/市值（批量）
- 理杏仁 API → ROE/毛利率/营收增速（逐股）
- 同花顺 EPS 预测 → CAGR 补充（逐股）

结果：PE/ROE/CAGR 回填率达到 85-100%。

### 决策 5：自动推荐赛道

用户问「其他板块需要手动输入还是能自动挑？」

**方案**：五维评分系统

| 维度 | 初始权重 | 调整后 | 调整原因 |
|------|----------|--------|----------|
| 价格强度 | 40% | 20% | 当日涨跌幅权重过重，追涨导向 |
| 资金热度 | 30% | 35% | 成交额是最可靠的关注度指标 |
| 上涨广度 | 20% | 30% | 板块性机会 > 独苗拉指数 |
| 持续性 | 10% | 15% | 连续上榜比一日游更有价值 |
| 容量加分 | - | +5(>1000亿) | 大板块标的丰富，更适合 BOM 拆解 |

用户反馈「消费电子最好能包含在内」→ 加入容量加分后从第 8 升至第 6。

### 决策 6：板块重合度

用户担心贵金属/小金属、元件/通信设备/消费电子/半导体之间有成分股重叠。

**结论**：同花顺行业分类互斥，每只股票只归一个板块。不需要合并或舍弃。它们是同一大产业链的不同环节（半导体→元件→通信/消费电子），分开分析更有价值。

### 决策 7：每日批量分析

用户要求每日自动分析 Top 5 赛道。

**实现**：`--daily` 命令 → 推荐 Top 5 → 跳过近 3 天已分析 → 逐个跑全流程。

## 三、实施步骤

| # | 步骤 | 产出 |
|---|------|------|
| 1 | 出方案 | Plan 模式 → 设计文档 → 用户批准 |

| 4 | Prompt | `stage1_chain.txt` + `stage2_leaders.txt` |
| 5 | 报告 | `report.py` — 5 节 Markdown + Obsidian frontmatter |
| 6 | 主管线 | `run.py` — argparse + 两阶段 LLM + 数据注入 + 后处理 |
| 7 | 端到端测试 | 跑通「AI算力」全流程 |
| 8 | 财务补全 | `_enrich_financials()` 替换名称校验，回填 PE/ROE/CAGR |
| 9 | 自动推荐 | `--suggest` / `--auto` / `--daily` + 五维评分 + 行业快照 |
| 10 | 批量验证 | `--daily` 一次跑通 5 个赛道，2 处幻觉修正 |

### 遇到的坑

1. **import 路径冲突**：`sys.path` 中 `daily_review/` 覆盖了 `bom_analyzer/` 的 `models.py`，修复为 `from bom_analyzer.models import ...`
2. **JSON null 导致 TypeError**：LLM 输出的 `"pe_ttm": null` 覆盖了 `dict.get(key, 0)` 的默认值，修复为 `dict.get(key) or 0`
3. **akshare 超时挂死**：成分股查询接口不稳定，最终基于同花顺行业分类规则得出结论（无需 API）

### 使用方法

```powershell
python bom_analyzer/run.py --suggest    # 查看今日推荐赛道
python bom_analyzer/run.py --auto       # 一键分析 Top 1
python bom_analyzer/run.py --daily      # 批量分析 Top 5
python bom_analyzer/run.py "行业名"     # 手动分析指定行业
python bom_analyzer/run.py --list       # 查看已分析行业
```
