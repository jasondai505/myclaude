# 边际变化跟踪

持续追踪一家上市公司关键投资逻辑的边际变化，将多源信息整理为可追溯的变化链，帮助投资者及时捕捉基本面拐点。

## 适用场景

- 长期跟踪某家公司的某条投资逻辑（如产能扩张进度、订单转化率、新产品定价）
- 希望将路演纪要、点评、公告中的分散信息汇聚到一张动态表格中
- 管理多标的跟踪任务，形成有据可查的变化记录

## 输入

- 公司名称与股票代码
- 需要跟踪的核心主题（如"海外收入占比"、"产能利用率"、"新签订单"）
- 可选：指定时间窗或信息来源范围

## 输出

以 Markdown 表格呈现的边际变化记录，每条记录包含：


| 字段   | 说明                      |
| ---- | ----------------------- |
| 日期   | 信息披露日期                  |
| 主题   | 所属跟踪主题                  |
| 变化方向 | 边际向好 / 边际下滑 / 符合预期      |
| 变动内容 | 具体数值或事件描述               |
| 变动原因 | 与上次记录的具体对比（注明前次数值与本次数值） |
| 来源   | 信息出处（纪要/公告/点评）          |
| 对比来源 | 作为对比基准的历史记录 ID          |


## 核心设计

每条变化记录必须与历史记录形成可追溯的对比链——不允许孤立记录，必须标明"相较何时何值，本次变成了什么"。首次记录标注为"首次记录"。

数据来源：路演纪要、卖方点评、公司公告、私域知识库，多源交叉验证。
